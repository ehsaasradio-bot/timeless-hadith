What We Are Building

Platform Overview

Alhuda 2.0 reuses the entire Timeless Hadith codebase with a fresh identity, new independent Supabase database, and a separate Cloudflare deployment.

⑂

Codebase Fork

Clone timeless-hadith into a new alhuda-2-0 GitHub repo — inheriting all features, pages, and logic instantly with zero rebuild.

🗄

New Supabase Database

Fresh PostgreSQL project with the same proven schema. You provide credentials; Claude handles migration, RLS, and auth setup.

☁

Cloudflare Pages

Dedicated CF Pages project — auto-deploys on every push to main. Free global CDN and SSL from day one.

◈

Complete Rebrand

New name, logo, color palette, favicon, and all body copy updated throughout — no traces of Timeless Hadith remain.

✦

AI Hadith Search

GPT-4o-mini hybrid search combining full-text tsvector and semantic embeddings. Fast, grounded, and hallucination-free.

⊕

Custom Domain + SSL

New domain configured on Cloudflare with automatic SSL and enterprise-grade CDN — configured and live on launch day.

Technology

Full Stack

Proven, cost-effective infrastructure — fully independent from Timeless Hadith from day one.

Hosting

Cloudflare Pages

Edge hosting with automatic deployment from GitHub. Global CDN, free SSL, and Cloudflare's DDoS protection at zero cost on the free tier.

Database

Supabase PostgreSQL

New Alhuda 2.0 Supabase project. Same schema as Timeless Hadith. Row Level Security, REST API, and real-time subscriptions.

AI

OpenAI GPT-4o-mini

Hybrid search and content generation. Semantic embeddings via text-embedding-3-small. User provides API key before Phase 8.

Auth

Supabase Auth

Email/password auth with JWT sessions. Replaces the Firebase Auth from Timeless Hadith. No extra vendor dependency.

Voice

Web Speech API

Browser-native voice reading — no API costs, no third-party service. Works on all modern browsers across mobile and desktop.

CI/CD

GitHub Actions

Validation pipeline runs before every Cloudflare deployment. Prevents broken code from reaching production.

01
Phase

Supabase Project Creation

New database — credentials must be provided before this phase begins

Day 1–2
feat(db)
1

Create new Supabase project

Go to supabase.com → New Project. Name it alhuda-2. Choose the region closest to your audience. Copy the Project URL and Anon key.

2

Provide credentials to Claude

Share: Supabase Project URL, Anon (public) Key, and Service Role Key. These are stored only in Cloudflare Pages env vars — never committed to the codebase.

3

Run schema migration

Export SQL schema from Timeless Hadith Supabase and import into the new project. Tables: hadiths, blog_posts, bookmarks, categories, profiles.

4

Configure Row Level Security

Public read on hadiths and published blog_posts. Auth-required write for bookmarks and profiles. No anonymous write access.

5

Enable Supabase Auth & remove Firebase

Configure email/password authentication. Delete js/firebase-auth.js and replace with js/supabase-auth.js. Remove Firebase SDK references from all pages.

Required From You
  • Supabase Project URL
  • Supabase Anon (public) Key
  • Supabase Service Role Key
  • Migrate TH data? Yes / No
Tables Created
  • hadiths (7,277+ rows)
  • blog_posts
  • bookmarks
  • categories
  • profiles

04
Phase

Cloudflare Pages Deployment

New Cloudflare project auto-deploys from the Alhuda 2.0 repo on every push to main

Day 3–4
chore(deploy)
1

Create Cloudflare Pages project

Cloudflare Dashboard → Pages → Create a Project → Connect to Git → Select alhuda-2-0 repo. Production branch: main. Build command: none (static site).

2

Add environment variables

In Settings → Environment Variables add: SUPABASE_URL, SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY, OPENAI_API_KEY (when available).

3

Test initial deployment

Verify the auto-generated alhuda-2-0.pages.dev URL loads correctly. Open browser console and confirm no errors. Verify Supabase queries return data.

4

Configure custom domain

CF Pages → Custom Domains → Add domain. Update DNS A/CNAME at your registrar to point to Cloudflare Pages. SSL certificate auto-provisions within 5 minutes.

Required From You
  • Cloudflare account access
  • GitHub OAuth for CF Pages
  • Domain name for Alhuda 2.0
  • DNS registrar credentials
Env Vars to Add in CF
  • SUPABASE_URL
  • SUPABASE_ANON_KEY
  • SUPABASE_SERVICE_ROLE_KEY
  • OPENAI_API_KEY
chore(deploy): cloudflare pages configured and live at alhuda domain

Phase

Data Migration

Import the full 7,277-hadith dataset into the new Alhuda 2.0 Supabase database

Day 4–6
feat(data)
1

Export from Timeless Hadith Supabase

Use Supabase Dashboard or pg_dump to export the hadiths table (7,277 rows) as a SQL dump or CSV. Also export categories and blog_posts if desired.

2

Import into Alhuda 2.0 Supabase

Use Supabase SQL Editor or psql to run the import. Verify row counts match. Check encoding — all Arabic text must round-trip correctly in UTF-8.

3

Rebuild full-text search indexes

Run UPDATE hadiths SET fts = to_tsvector('english', text_en) || to_tsvector('arabic', text_ar) to rebuild tsvector search columns for both languages.

4

Verify on the live site

Open Alhuda 2.0 in the browser. Confirm hadiths load on the home page, categories return results, and search returns relevant hadith cards from the new database.

feat(data): hadith data migrated and search indexes rebuilt

Phase

AI-Powered Search Integration

Connect OpenAI for hybrid semantic + full-text hadith search — fast, grounded, hallucination-free

Day 9–11
feat(ai)
1

Connect OpenAI API key

User provides the OpenAI API key. Add it to Cloudflare Pages environment variables. Configure model as gpt-4o-mini for cost efficiency.

2

Generate and store embeddings (optional)

For full semantic search, generate text-embedding-3-small embeddings for each hadith and store in a Supabase pgvector column. Enables meaning-based retrieval beyond keywords.

3

Wire hybrid search endpoint

Via Supabase Edge Function: run full-text tsvector query first, then re-rank top results using semantic vector similarity. Return the top 10 most relevant hadiths.

4

Test search quality

Run test queries in both English and Arabic. Verify results are relevant, fast (under 2 seconds), and strictly grounded in database content — no AI-invented hadiths.

Required From You
  • OpenAI API key
  • Enable embeddings? Yes / No
  • Monthly OpenAI budget limit
Search Architecture
  • tsvector full-text (primary)
  • pgvector semantic (re-rank)
  • Response target: under 2s
feat(ai): openai hybrid search integrated for alhuda-2-0
