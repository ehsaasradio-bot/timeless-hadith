AI Search Deep Dive

How grounded AI search works: hybrid full-text + semantic retrieval without hallucinations.

The Problem

Users want to ask natural language questions about Islam, and get instant, authoritative answers. Traditional keyword search misses nuance; pure LLMs hallucinate.

The Solution: Hybrid RAG Pipeline

  1. Query Vectorization: User question → OpenAI text-embedding-3-small (1536 dims)
  2. Vector Search: Query vector → Supabase pgvector similarity search (cosine distance) → Top 5 hadiths by semantic similarity
  3. Full-Text Fallback: If vector results are weak, Supabase tsvector full-text search adds keyword matches
  4. Grounding: Top results concatenated into a RAG context prompt
  5. Generation: gpt-4o-mini synthesizes a natural answer grounded exclusively in the context
  6. Citation: Answer includes hadith references (narrator, book, category)

OpenAI Embeddings

Model: text-embedding-3-small

Dimensions: 1536

Cost: $0.02 per 1M tokens (~$2 to embed all 7,277 hadiths)

Speed: Batch processing, ~100ms per request

Quality: Optimized for semantic relevance on religious texts
