Critical Issues

2 issues
Critical
ReferenceError: dot is not defined — Pagination Completely Broken
/category (all book pages) · line 808–815 · fires on every page render

Every time a category detail page renders, updatePagination() throws 49+ consecutive ReferenceErrors. The variable var dot is declared inside an IIFE but V8 fails to hoist it within this specific scope, so dot.className on the next line throws immediately. This means pagination dots never render on any hadith listing page.

Fix — rename the variable

In category.html, inside updatePagination(), rename dot → btn everywhere in the function body. This sidesteps the hoisting conflict entirely.

var btn = document.createElement('button');
btn.className = 'pagination-dot';
btn.setAttribute('aria-label', ...);

Critical
ReferenceError: i is not defined — Cookie Script Broken on About Page
/about · line 740–741 · stray variable in cookie consent block

The cookie consent script on about.html references a variable i that was never declared. This causes the entire cookie consent block to fail silently — the consent banner never fires correctly, which is a PDPL/privacy compliance issue since the site is targeted at Saudi users.

Fix — remove stray i

On line 741 of about.html, remove the orphaned i character. It was accidentally left during an edit. The cookie block should read: if (!store.get('th_cookie_consent')) {

High Priority Issues

High
All Canonical URLs Point to Wrong Domain
All pages · canonical href="https://timelesshadith.com/…" · should be timeless-hadith.pages.dev

Every page has <link rel="canonical"> pointing to timelesshadith.com instead of the live deployment at timeless-hadith.pages.dev. Google will treat the pages on the actual live domain as duplicates and may deprioritise them entirely. Additionally, categories and category pages still have .html extensions in their canonical URLs, but Cloudflare Pages strips those — causing another canonical mismatch.

Fix — update all canonicals to match actual live URL

/index.html → https://timeless-hadith.pages.dev/
/categories.html → https://timeless-hadith.pages.dev/categories
/category.html → set dynamically via JS to https://timeless-hadith.pages.dev/category?cat=…
/about.html → https://timeless-hadith.pages.dev/about

All pages · favicon-32.png, favicon-16.png, apple-touch-icon.png, manifest.json not found

The HTML references four icon files (favicon-32.png, favicon-16.png, apple-touch-icon.png, and manifest.json) that do not exist in the repository. High
changePage() Not in Global Scope — Prev/Next Buttons Dead
/category · onclick="changePage(-1)" · function inside IIFE, not accessible

The Prev and Next navigation buttons on every category page use onclick="changePage(-1)" and onclick="changePage(1)". However, the changePage() function is defined inside an IIFE (function(){…})() and is therefore not accessible from inline HTML event handlers. Both buttons silently do nothing when clicked — confirmed with typeof window.changePage === 'undefined'.

Fix — expose function globally

Change function changePage(dir) {…} to window.changePage = function(dir) {…} inside the IIFE, or use addEventListener on the buttons instead of inline onclick attributes.

Medium Priority Issues

Medium
Category Page Doesn't Respond to ?id= Parameter
/category?id=faith → shows "Category not found" · only ?cat=book-N works

Navigating to /category?id=faith or /category?id=prayer returns "Category not found". The page only accepts the ?cat=book-N format (e.g. ?cat=book-2). This breaks any external links using the more readable ?id= format, and makes URLs non-descriptive for SEO.

Fix — support slug-based URLs

Add support for ?cat=faith by mapping topic slugs to book IDs in the JS, or standardise on one URL format and ensure all internal links use it consistently.

Medium
Category & Terms Pages Have Duplicate if Block in Cookie Script
/terms · lines 656–658 · duplicate if (!store.get…) blocks back to back

terms.html has two consecutive identical if (!store.get('th_cookie_consent')) checks. The second block overwrites any state set by the first, causing the cookie banner to potentially display twice or behave unexpectedly. This is a copy-paste error from a previous edit.

Fix — remove duplicate block

Delete the second if (!store.get('th_cookie_consent')) {…} block on line 658 of terms.html.

Medium
OG & Canonical URLs Use Wrong Domain on All Pages
All pages · og:url, og:image, twitter:image → timelesshadith.com (not live domain)

All og:url and social meta tags reference timelesshadith.com — a domain that may not be live. When the page is shared on social media, link previews will show the wrong URL. Combined with the 404 OG image, social sharing produces broken, unbranded previews across all pages.

Fix — update og:url and og:site_name across all pages

Replace https://timelesshadith.com/ with https://timeless-hadith.pages.dev/ in all og:url, og:image, and twitter:image tags. Also update the JSON-LD "logo" and "url" fields in the structured data blocks.

Medium
Category Detail Page Has Static Canonical (Not Dynamic)
/category · canonical always = timelesshadith.com/category.html · ignores ?cat= param

The category detail page uses a hardcoded <link rel="canonical" href="https://timelesshadith.com/category.html">. Since this page renders different content based on the ?cat= URL parameter, every book page (book-1 through book-97) has the exact same canonical URL — telling Google they are all the same page.

Fix — set canonical dynamically via JavaScript

In the page-init JS, after parsing the ?cat= param, update the canonical tag:
document.querySelector('link[rel="canonical"]').href = 'https://timeless-hadith.pages.dev/category?cat=' + catId;

Low
No sitemap.xml or robots.txt Found
Site root · sitemap.xml · robots.txt

The site has no sitemap.xml or robots.txt file. Without a sitemap, Google's crawler must discover all pages by following links — slower and less reliable for a content-heavy hadith site with 97 books.

Fix — add sitemap.xml and robots.txt

Create a minimal robots.txt pointing to the sitemap. Generate a sitemap.xml listing all static pages (/, /categories, /about, /privacy, /terms, /bookmarks) plus dynamic category URLs.

Low
JSON-LD Schema Missing on Category & Bookmark Pages
/category, /bookmarks · no structured data

The homepage has a WebSite schema, but category detail and bookmarks pages have no JSON-LD structured data. Adding BreadcrumbList or CollectionPage schema to category pages can help Google understand the content hierarchy and generate rich snippets.

Fix — add BreadcrumbList schema to category pages

Inject a BreadcrumbList JSON-LD block dynamically when the category page loads, referencing: Home → Categories → [Book Name].
