🔴 Critical Issues
C1 — categories.html: SyntaxError crashes entire Browse page
Critical
📄 /categories — Line 857–860
The applySearch() function body is truncated — it begins mid-expression with a filter callback fragment but is missing the wrapping filteredCats = allCats.filter(function(cat){ line. The browser throws SyntaxError: Unexpected token ')' at line 858, which crashes the entire inline script. As a result: no category cards are rendered, and the search input does nothing.
// BROKEN — function body starts mid-expression function applySearch(q) { (PILLS[cat.slug]||[]).join(' ').toLowerCase().indexOf(q) !== -1 ); }); }
✅ Fix
Restore the missing top of the function body — the filteredCats = allCats.filter(function(cat){ return ... }) wrapper and the category title/pill search conditions.
C2 — categories.html: TH.categories is an Object, not an Array
Critical
📄 /categories — Line 886–887
TH.categories returns a plain object keyed by index ({"0":{...}, "1":{...}}). The categories page assigns it to allCats and immediately calls allCats.slice(), which is undefined on plain objects. This would silently fail or throw a TypeError when trying to render or filter categories.
allCats = TH.categories; // Object, not Array filteredCats = allCats.slice(); // TypeError: allCats.slice is not a function // Fix: convert object values to array first allCats = Object.values(TH.categories); filteredCats = allCats.slice();
✅ Fix
Replace allCats = TH.categories with allCats = Object.values(TH.categories) on line 886.

High

The "Most Viewed" horizontal pill bar on the homepage links to slugs like faith-belief, character-conduct, prayer-worship, etc. However, all Supabase data is keyed by book number slugs (book-1 through book-97). ✅ Fix
Update the "Most Viewed" pills in index.html to use the correct book slugs (e.g., book-2 for Belief, book-3 for Knowledge) — or map the display labels to real slugs dynamically from TH.categories.

🟡 Medium Issues
M1 — Canonical tag and og:url hardcoded to wrong domain
Medium
📄 All pages — <head> meta tags
Every page has its <link rel="canonical"> and og:url hardcoded to https://timelesshadith.com/ (the root homepage URL). This means Google sees every single page as a duplicate of the homepage. Category pages, about, privacy, and terms all point to the same canonical, which will cause serious SEO de-indexing issues.
<link rel="canonical" href="https://timelesshadith.com/" /> ← same on ALL pages <meta property="og:url" content="https://timelesshadith.com/" /> <!-- Fix: use page-specific canonical --> <link rel="canonical" href="https://timelesshadith.com/categories" />
✅ Fix
Each page needs its own unique canonical URL. Update all HTML files to set the correct page-specific canonical and og:url. The category detail page already has a JS-based dynamic canonical — apply the same pattern to all pages.
M2 — Sitemap uses .html extensions; Cloudflare strips them
Medium
📄 sitemap.xml
The sitemap lists URLs as /categories.html, /about.html, etc. However, Cloudflare Pages automatically removes the .html extension — the real accessible URLs are /categories, /about, /privacy, etc. Search engines indexing the sitemap will encounter redirects or mismatches.
<loc>https://timelesshadith.com/categories.html</loc> ← redirects <loc>https://timelesshadith.com/categories</loc> ← correct
✅ Fix
Remove all .html extensions from sitemap.xml URLs. Also ensure the sitemap domain matches the live domain consistently.
M3 — "Daily Hadith" CTA links to anchor, not a daily hadith feature
Medium
📄 / (Homepage) — Hero section
The hero button labelled "Daily Hadith" links to /#featured, which simply scrolls down to the Featured Hadiths carousel. It does not show a date-specific hadith or any dedicated "hadith of the day" experience. The label creates a false expectation for users who click it expecting a daily content feature.
✅ Fix (2 options)
Option A: Rename the button to "Featured Hadiths" to accurately describe what it does. Option B: Build a true daily hadith feature that selects a hadith by day-of-year and links to a dedicated /daily page.
