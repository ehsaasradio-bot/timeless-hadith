H1 — Category pagination dots still broken: ReferenceError: dot is not defined Remains
Page: /category?cat=book-N
High

Despite commit b0f2030 adding var dot=document.createElement('button'); on line 808 inside the updatePagination() IIFE, the browser still throws ReferenceError: dot is not defined at line 809 (dot.className=...) on every render cycle. The pagination-dots container remains perpetually empty.

Confirmed via: window.onerror capture, Element.prototype.appendChild patch (zero calls), and direct setTimeout wrapper — the error fires on every renderPage() call.

Effect: No pagination dots visible. Users cannot see which page they are on. Previous/Next buttons are the only navigation (and those are also broken — see H2).

The var dot fix is present in the deployed source at the correct position, but the runtime behaviour suggests a deeper issue with this specific IIFE pattern. Recommended fix: rewrite the loop to avoid the IIFE entirely — declare dot outside the IIFE or use a named helper function.
// Current (broken — var dot hoisting not working as expected):
for(var i=0;i<pages;i++){
  (function(idx){
    var dot=document.createElement('button');  // line 808
    dot.className='p-dot'...;                  // line 809 — throws!
  })(i);
}

// Recommended fix:
for(var i=0;i<pages;i++){
  (function(idx){
    var btn=document.createElement('button');
    btn.className='p-dot'+(idx===currentPage?' active':'');
    btn.setAttribute('role','listitem');
    btn.setAttribute('aria-label','Page '+(idx+1));
    if(idx===currentPage) btn.setAttribute('aria-current','page');
    btn.onclick=function(){ if(idx!==currentPage){ currentPage=idx; renderPage(); } };
    dots.appendChild(btn);
  })(i);
}
H2 — Category Previous/Next buttons broken: changePage not in global scope New
Page: /category?cat=book-N
High

The pagination Previous and Next buttons use onclick="changePage(-1)" and onclick="changePage(1)" inline HTML attributes. These require changePage to be available on the global window object. However, changePage is defined inside the page's IIFE ((function() { ... })()) and is never exposed globally.

Confirmed: typeof window.changePage === 'undefined' in browser console. Clicking Previous/Next throws Uncaught ReferenceError: changePage is not defined.

Effect: Multi-page categories (most books have 9+ hadiths) cannot be navigated beyond the first page.

// In category.html IIFE — changePage is local, not global:
(function() {
  function changePage(dir) { ... }  // ← not accessible via onclick=""
  // ...
})();

// Fix option 1 — expose on window:
window.changePage = function(dir) { ... };

// Fix option 2 — use addEventListener instead of onclick attributes:
document.getElementById('btn-prev').addEventListener('click', function(){ changePage(-1); });
document.getElementById('btn-next').addEventListener('click', function(){ changePage(1); });

M2 — All canonical and og:url tags point to wrong domain Remains
Page: All pages (global)
Medium

Every page has its <link rel="canonical"> and <meta property="og:url"> hardcoded to timelesshadith.com instead of timeless-hadith.pages.dev (or the intended production domain). This tells search engines and social platforms that the canonical version of the page is at a different URL, potentially harming SEO and causing duplicate-content signals.

Examples found:

  • Homepage: https://timelesshadith.com/
  • Categories: https://timelesshadith.com/categories.html
  • Category detail: https://timelesshadith.com/category.html?cat=book-2
  • About: https://timelesshadith.com/about.html
Update canonical and og:url to use the correct live domain on all pages. If the site will move to a custom domain, set them to that domain now.
