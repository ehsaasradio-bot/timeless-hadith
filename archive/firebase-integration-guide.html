1

Paste your Firebase config into js/firebase-auth.js

The file is already created. Open it and replace the placeholder firebaseConfig object (lines 48–56) with your real values from the Firebase Console.

Where to find it: Firebase Console → ⚙️ Project Settings → Your apps → SDK setup and configuration → Config

js/firebase-auth.js — lines 48–56 (replace this) 
// ── BEFORE (placeholders) ──
const firebaseConfig = {
  apiKey:            "YOUR_API_KEY",
  authDomain:        "YOUR_PROJECT_ID.firebaseapp.com",
  projectId:         "YOUR_PROJECT_ID",
  storageBucket:     "YOUR_PROJECT_ID.appspot.com",
  messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
  appId:             "YOUR_APP_ID"
};

// ── AFTER (your real values from Firebase Console) ──
const firebaseConfig = {
  apiKey:            "AIzaSyC_abc123defXXXXXXXXXXX",       // ← your real key
  authDomain:        "timeless-hadith.firebaseapp.com",
  projectId:         "timeless-hadith",
  storageBucket:     "timeless-hadith.appspot.com",
  messagingSenderId: "123456789012",
  appId:             "1:123456789012:web:abc123def456"
};

Note on API key exposure: It is safe to include your Firebase API key in client-side code. Firebase API keys are not secret — they identify your project to Firebase, not grant admin access. Security is enforced by Firebase Security Rules and Authorised Domains. Do NOT commit service account credentials or Admin SDK keys.

2

Update the Content-Security-Policy in index.html

Your current CSP has connect-src 'none' — this blocks all network requests, including Firebase's auth API calls. Replace the entire CSP meta tag with the version below.

Location: index.html → <head> → around line 31–32

❌ Remove this (current CSP):

index.html — <head> (REMOVE)
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self' 'unsafe-inline'
               https://accounts.google.com https://fonts.googleapis.com;
               style-src 'self' 'unsafe-inline' https://fonts.googleapis.com
               https://fonts.gstatic.com; img-src 'self' data: https:;
               font-src 'self' https://fonts.gstatic.com;
               connect-src 'none';" />

Paste this (updated CSP):

index.html — <head> (PASTE THIS) 
<meta http-equiv="Content-Security-Policy"
      content="
        default-src  'self';

        script-src   'self' 'unsafe-inline'
                     https://www.gstatic.com
                     https://apis.google.com
                     https://accounts.google.com
                     https://fonts.googleapis.com;

        style-src    'self' 'unsafe-inline'
                     https://fonts.googleapis.com
                     https://fonts.gstatic.com;

        img-src      'self' data: https: blob:;

        font-src     'self' https://fonts.gstatic.com;

        connect-src  'self'
                     https://identitytoolkit.googleapis.com
                     https://securetoken.googleapis.com
                     https://www.googleapis.com
                     https://*.firebaseio.com;

        frame-src    https://accounts.google.com
                     https://*.firebaseapp.com;
      " />

Why each domain?
www.gstatic.com — serves the Firebase SDK JS files via CDN
apis.google.com — Google API gateway used by Firebase Auth
identitytoolkit.googleapis.com — Firebase Auth REST API (sign-in/out)
securetoken.googleapis.com — issues and refreshes ID tokens
*.firebaseapp.com — Firebase popup/redirect auth frame
img-src blob: — needed for avatar images loaded via object URLs

3

Replace the GIS script tag on every HTML page

Each page currently loads Google Identity Services. Swap it for the Firebase auth module. This is the only change needed on sub-pages.

Location on each page: Just before </body>, after js/app.js

❌ Remove this line (on every page):

Every HTML page — before </body> (REMOVE)
<script src="https://accounts.google.com/gsi/client" async defer></script>

Add this line in its place (on every page):

Every HTML page — before </body> (ADD) 
<script type="module" src="js/firebase-auth.js"></script>

Full context — what the bottom of each page should look like:

Every HTML page — before </body> (final state) 
  <!-- ── App scripts ── -->
  <script src="js/data.js"></script>
  <script src="js/app.js"></script>
  <script type="module" src="js/firebase-auth.js"></script>
</body>
</html>

Why type="module"? Firebase v10 uses ES module syntax (import / export). The type="module" attribute enables this in the browser. Modules are deferred automatically — no async / defer needed. They also run in strict mode and have their own scope, preventing variable leaks.

4

Test locally before deploying

Firebase Auth requires a proper HTTP server — it won't work when opening HTML files directly (file:// URLs). Use one of these:

Terminal — pick any of these local servers 
# Option A — Python (usually pre-installed)
python3 -m http.server 8000

# Option B — Node.js
npx serve .

# Option C — VS Code Live Server extension
# Right-click index.html → "Open with Live Server"

Then open http://localhost:8000 in your browser. Make sure localhost is in your Firebase Authorised Domains list (it usually is by default).

Checklist

Page loads without console errorsNo Firebase init errors

5

Deploy to GitHub Pages

Before pushing, make sure your GitHub Pages domain is in Firebase's Authorised Domains list.

Firebase Console → Authentication → Settings → Authorised domains
localhost                            ← already there by default
YOUR-USERNAME.github.io             ← add this
timelesshadith.com                  ← add this (if using custom domain)

You're done. No Firebase Hosting, no server, no build pipeline required. The modular CDN imports work directly in the browser. GitHub Pages serves static files — Firebase Auth does all the heavy lifting via its own infrastructure.

★

Using getCurrentUser() in your other scripts

The function is exposed on window so any script on the page can call it — no imports needed from non-module scripts like app.js.

Any script (app.js, bookmarks.js, etc.) 
// Get the current user anywhere
const user = window.getCurrentUser();

if (user) {
  // User is signed in
  console.log(user.uid);           // "abc123xyz..." (use as Firestore key)
  console.log(user.displayName);  // "Syed"
  console.log(user.email);        // "syed@example.com"
  console.log(user.photoURL);     // "https://lh3.googleusercontent.com/..."
} else {
  // No user signed in
}

// Listen for auth state changes (fired on sign-in and sign-out)
document.addEventListener('th:login',  (e) => {
  const user = e.detail.user;
  // e.g. load user's bookmarks from localStorage
});

document.addEventListener('th:logout', () => {
  // e.g. clear bookmarks from UI
});
