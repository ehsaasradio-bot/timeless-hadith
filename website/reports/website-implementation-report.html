Next Steps & Recommendations

HIGH PRIORITY
Push commits to GitHub remote

Branch main is 2 commits ahead of origin/main. Once approved, push with git push origin main to trigger Cloudflare Pages auto-deploy.

HIGH PRIORITY
Connect frontend to Supabase 7,277 hadith database

The frontend currently uses static js/data.js. Migration to live Supabase queries via supabase-data.js is the most impactful next step for content depth.

HIGH PRIORITY
Migrate auth from Firebase to Supabase

js/firebase-auth.js is still in use. Migrating to js/supabase-auth.js consolidates the stack and removes Firebase dependency.

MEDIUM PRIORITY
Add OG tags and JSON-LD to bookmarks.html, privacy.html, terms.html

These three pages are missing full Open Graph metadata and structured data. Low-effort improvement with measurable SEO/social sharing benefit.

MEDIUM PRIORITY
Choose social media automation tool

At project completion, decide between Buffer ($6/mo, simplest) or Make.com ($9/mo, most flexible) for auto-posting to Facebook, Twitter/X, Instagram, YouTube, and LinkedIn.

MEDIUM PRIORITY
Enable AI semantic search (OpenAI embeddings)

The hybrid search system (full-text + vector) is architected in js/ai-search.js but requires an OpenAI API key. Adding this will dramatically improve search relevance.

LOW PRIORITY
Add og:image meta to category.html dynamically

The category page already dynamically updates og:title and og:description via JS. The og:image could also be updated per category for richer social sharing.

LOW PRIORITY
Run Lighthouse audit on live Cloudflare Pages URL

Target: Performance 90+, Accessibility 90+, SEO 95+. The SEO foundation is strong — a live audit will surface any runtime issues not visible in static analysis.
