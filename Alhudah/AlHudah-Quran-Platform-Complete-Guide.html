Database Design

Supabase PostgreSQL stores all Quran data: surahs, verses, translations, tafsir, and AI embeddings. The schema supports full-text search, vector similarity, and Row-Level Security.

Core Tables

Table 1: Surahs (114 chapters)

Column Type Description
id SERIAL PRIMARY KEY Unique identifier
number INTEGER UNIQUE 1-114 (unique, indexed)
name_arabic TEXT الفاتحة (with tashkeel)
name_english TEXT Al-Fatihah
meaning TEXT The Opening
revelation_type TEXT 'meccan' OR 'medinan'
total_ayahs INTEGER 1-286 verses
juz_start INTEGER Starting juz (1-30)
page_start, page_end INTEGER Madani mushaf pages
slug TEXT UNIQUE al-fatihah (URL-safe)
seo_title, meta_description TEXT Meta tags (60 & 160 chars)

Table 2: Ayahs (6,236 verses)

Column Type Description
id SERIAL PRIMARY KEY Unique identifier
surah_number INTEGER FK to surahs(number)
ayah_number INTEGER 1-286 within surah
ayah_key TEXT UNIQUE "1:1", "2:255" (canonical)
text_arabic TEXT Full Arabic with diacritics
text_english TEXT Sahih International (ID 131)
text_urdu TEXT Mufti Taqi Usmani (ID 234)
juz, hizb, page INTEGER Structural markers
sajda BOOLEAN Prostration marker
search_vector TSVECTOR Full-text search index
embedding VECTOR(1536) OpenAI Ada-002 embedding

  page_start INTEGER,

  seo_title TEXT,
  meta_description TEXT,

CREATE INDEX idx_surahs_number ON surahs(number);
CREATE INDEX idx_surahs_slug ON surahs(slug);

-- Create ayahs table
CREATE TABLE ayahs (
  id SERIAL PRIMARY KEY,
  surah_number INTEGER NOT NULL,
  ayah_number INTEGER NOT NULL,
  ayah_key TEXT UNIQUE NOT NULL,
  text_arabic TEXT NOT NULL,
  text_english TEXT,
  text_urdu TEXT,
  text_transliteration TEXT,
  juz INTEGER NOT NULL,
  hizb INTEGER,
  hizb_quarter INTEGER,
  page INTEGER,
  sajda BOOLEAN DEFAULT FALSE,
  sajda_type TEXT,
  search_vector TSVECTOR,
  embedding VECTOR(1536),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT fk_ayah_surah FOREIGN KEY (surah_number) REFERENCES surahs(number),
  CONSTRAINT uq_ayah_position UNIQUE(surah_number, ayah_number)
);

CREATE INDEX idx_ayahs_ayah_key ON ayahs(ayah_key);
CREATE INDEX idx_ayahs_surah_number ON ayahs(surah_number);
CREATE INDEX idx_ayahs_juz ON ayahs(juz);
CREATE INDEX idx_ayahs_search_vector ON ayahs USING GIN(search_vector);
CREATE INDEX idx_ayahs_embedding_ivfflat ON ayahs
  USING ivfflat (embedding vector_cosine_ops)
  WITH (lists = 100);

JavaScript Modules

data-layer.js: Abstracts all Supabase calls

// Fetch all surahs
async function getSurahs() {
  const { data, error } = await supabase
    .from('surahs')
    .select('*')
    .order('number');
  return data;
}

// Fetch ayahs by surah
async function getAyahs(surahNumber) {
  const { data, error } = await supabase
    .from('ayahs')
    .select('*')
    .eq('surah_number', surahNumber)
    .order('ayah_number');
  return data;
}

// Fetch tafsir for a verse
async function getTafsir(tafsirId, ayahKey) {
  const { data, error } = await supabase
    .from('tafsir_entries')
    .select('*')
    .eq('tafsir_id', tafsirId)
    .eq('ayah_key', ayahKey)
    .single();
  return data;
}

theme.js: Light/dark toggle

function initTheme() {
  const saved = localStorage.getItem('alhudah-theme') || 'light';
  document.documentElement.setAttribute('data-theme', saved);
}

function toggleTheme() {
  const current = localStorage.getItem('alhudah-theme') || 'light';
  const next = current === 'light' ? 'dark' : 'light';
  localStorage.setItem('alhudah-theme', next);
  document.documentElement.setAttribute('data-theme', next);
  document.dispatchEvent(new CustomEvent('theme-changed', { detail: { theme: next } }));
}
Performance Tip: Use Intersection Observer for lazy-loading long surahs (Al-Baqarah has 286 verses). Load first 20, then load on scroll. Virtual scrolling not necessary for most surahs.

AI-Powered Search

Users ask natural language questions about the Quran. The system performs semantic search via OpenAI embeddings and full-text search via Supabase, then generates a grounded answer using GPT-4o-mini.

Search Pipeline

1. User Input

User types or speaks a question: "What does Quran say about patience?"

2. Validation

Check query length (5-1000 chars), rate limit (10 req/min), sanitize input.

3. Generate Embedding

Call OpenAI to convert query to 1536-dim vector.

4. Vector Search

Query Supabase pgvector with cosine similarity. Find top 20 semantically related verses.

5. Full-Text Search

Query Supabase tsvector for exact keyword matches. Find top 20 keyword-matched verses.

6. Merge & Rank

Combine vector and keyword results. Weight by relevance score. Take top 10.

7. Generate Answer

Call GPT-4o-mini with grounded context. System prompt ensures answers cite Quranic verses.

8. Return Response

Frontend renders AI answer + numbered verse cards with citations.

Cloudflare Worker Code

Why a Worker? OpenAI API key cannot be exposed to browser. Worker acts as secure proxy.

export default {
  async fetch(request, env) {
    if (request.method !== 'POST') {
      return new Response('Method not allowed', { status: 405 });
    }

    const { query } = await request.json();

    // Validate input
    if (!query || query.length < 5 || query.length > 1000) {
      return new Response(JSON.stringify({
        error: 'Invalid query length (5-1000 chars)'
      }), { status: 400 });
    }

    // Rate limit check
    const ip = request.headers.get('cf-connecting-ip');
    const rateLimitKey = `ratelimit:${ip}`;
    const count = await env.RATE_LIMIT_KV.get(rateLimitKey) || 0;
    if (count > 10) {
      return new Response(JSON.stringify({
        error: 'Rate limit exceeded (10 req/min)'
      }), { status: 429 });
    }
    await env.RATE_LIMIT_KV.put(rateLimitKey, parseInt(count) + 1, { expirationTtl: 60 });

    const startTime = Date.now();

    try {
      // 1. Generate embedding for query
      const embeddingRes = await fetch('https://api.openai.com/v1/embeddings', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${env.OPENAI_API_KEY}`,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          model: 'text-embedding-3-small',
          input: query,
        }),
      });
      const embeddingData = await embeddingRes.json();
      const queryVector = embeddingData.data[0].embedding;

      // 2. Call Supabase RPC for vector search
      const vectorSearchRes = await fetch(
        `${env.SUPABASE_URL}/rest/v1/rpc/vector_search_ayahs`,
        {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${env.SUPABASE_SERVICE_KEY}`,
            'Content-Type': 'application/json',
            'Prefer': 'count=exact',
          },
          body: JSON.stringify({
            query_embedding: queryVector,
            similarity_threshold: 0.7,
            match_count: 20,
          }),
        }
      );
      const vectorResults = await vectorSearchRes.json();

      // 3. Call Supabase RPC for full-text search
      const ftSearchRes = await fetch(
        `${env.SUPABASE_URL}/rest/v1/rpc/fts_search_ayahs`,
        {
          method: 'POST',
          headers: {
            'Authorization': `Bearer ${env.SUPABASE_SERVICE_KEY}`,
            'Content-Type': 'application/json',
            'Prefer': 'count=exact',
          },
          body: JSON.stringify({
            search_query: query,
            match_count: 20,
          }),
        }
      );
      const ftResults = await ftSearchRes.json();

      // 4. Merge results with weighted scoring
      const mergedResults = mergeAndRankResults(vectorResults, ftResults);
      const topVerses = mergedResults.slice(0, 10);

      // 5. Generate AI answer with grounded context
      const verseContext = topVerses
        .map(v => `${v.ayah_key}: ${v.text_english}`)
        .join('\n\n');

      const chatRes = await fetch('https://api.openai.com/v1/chat/completions', {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${env.OPENAI_API_KEY}`,
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          model: 'gpt-4o-mini',
          messages: [
            {
              role: 'system',
              content: `You are a knowledgeable Quranic guide. Answer questions about the Quran
using only the provided verses. Cite specific verse numbers (Surah:Verse format).
Never invent or hallucinate verses. Be respectful, accurate, and concise.`,
            },
            {
              role: 'user',
              content: `Based on these Quranic verses:\n\n${verseContext}\n\nAnswer this question: ${query}`,
            },
          ],
          temperature: 0.7,
          max_tokens: 500,
        }),
      });

      const chatData = await chatRes.json();
      const answer = chatData.choices[0].message.content;

      const duration = Date.now() - startTime;

      return new Response(JSON.stringify({
        answer,
        verses: topVerses,
        status: 'success',
        duration_ms: duration,
      }), {
        headers: { 'Content-Type': 'application/json' },
      });
    } catch (error) {
      return new Response(JSON.stringify({
        error: error.message,
        status: 'failed',
      }), { status: 500, headers: { 'Content-Type': 'application/json' } });
    }
  }
}
