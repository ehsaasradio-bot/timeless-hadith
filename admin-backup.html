Restore Site
⏪

Restore from Git

Roll back to any previous commit. Cloudflare redeploys automatically on push.

  1. Find the commit hash

    Run git log --oneline -20 to list recent commits.

  2. Checkout the version

    git checkout <hash> -- . — restores files without losing history.

  3. Commit & push

    git commit -m "restore: roll back to <hash>" then git push origin main.

Quick restore

Enter a commit hash or tag to generate the restore commands.

📦

Restore from ZIP

Unzip a local snapshot and copy files back to the project folder.

  1. Locate snapshot

    Find backup-YYYY-MM-DD.zip in your backups folder.

  2. Unzip to temp folder

    unzip backup-2026-04-03.zip -d restore-tmp/

  3. Copy files back

    cp -r restore-tmp/* /path/to/Haith/

  4. Commit restored state

    Run git add -A && git commit -m "restore: from zip backup"

🗄️

Restore Supabase Data

Re-import hadiths from a JSON export using the Supabase SQL editor or REST API.

  1. Open Supabase SQL Editor

    Go to supabase.com → SQL Editor

  2. TRUNCATE existing table

    TRUNCATE TABLE hadiths; — only if doing a full restore.

  3. Import JSON via script

    Use the supabase-import.js helper script (see Setup Guide).

Setup Guide — Activate Scheduled Backups
🐧

Mac / Linux (crontab)

Open Terminal and edit your cron schedule with crontab -e, then add:

# Daily Git push at 02:00 0 2 * * * cd ~/Projects/Haith && git add -A && git commit -m "chore: auto-backup $(date +\%F)" --allow-empty && git push origin main # Weekly ZIP snapshot every Sunday at 03:00 0 3 * * 0 cd ~/Projects/Haith && zip -r ~/Backups/hadith-backup-$(date +\%F).zip . --exclude=".git/*" && ls -tp ~/Backups/ | grep -v '/$' | tail -n +6 | xargs -I {} rm -- ~/Backups/{} # Monthly Supabase export on 1st at 04:00 0 4 1 * * curl "https://dwcsledifvnyrunxejzd.supabase.co/rest/v1/hadiths?select=*&limit=10000" -H "apikey: YOUR_ANON_KEY" -o ~/Backups/hadiths-$(date +\%F).json
💡

The tail -n +6 | xargs rm part automatically deletes old ZIPs keeping only the 5 most recent.

🪟

Windows (Task Scheduler)

Open Task Scheduler → Create Basic Task → set the trigger and action.

  1. Daily Git push

    Program: cmd.exe
    Args: /c cd C:\Projects\Haith && git add -A && git commit -m "auto-backup" --allow-empty && git push origin main

  2. Weekly ZIP snapshot

    Use a PowerShell script:
    Compress-Archive -Path C:\Projects\Haith -Destination "C:\Backups\hadith-$(Get-Date -f yyyy-MM-dd).zip"

🤖

Cowork Scheduled Task

Use the built-in Cowork scheduler to trigger backups from this app automatically.

  1. Daily Git push task

    Task ID: hadith-git-backup
    Cron: 0 2 * * * (2am daily)

  2. Weekly ZIP snapshot task

    Task ID: hadith-zip-snapshot
    Cron: 0 3 * * 0 (3am every Sunday)

  3. Create tasks

    Click the button below — tasks are created in Cowork's scheduler and run automatically.

Backup Strategy Overview
Layer What is backed up Where stored Frequency Recovery time
🗂️ Git All HTML, JS, CSS, config files GitHub + Cloudflare Every commit (daily auto) ~2 min (git checkout + push)
💾 ZIP Snapshot Entire project folder (excl. .git) Local ~/Backups Weekly (Sunday 03:00) ~5 min (unzip + git commit)
🗄️ Supabase Export Hadith database (all rows, all books) Local JSON + CSV file Monthly (1st at 04:00) ~15 min (reimport via SQL)
☁️ Cloudflare Deployed site + Pages Functions Cloudflare CDN (auto) On every git push Automatic (no action needed)
🛡️

Rule of 3-2-1: Keep 3 copies of data, on 2 different media, with 1 offsite. This setup satisfies all three — Git (local + GitHub remote) + Cloudflare CDN gives you 3 locations automatically.
