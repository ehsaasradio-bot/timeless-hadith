Timeless Hadith is a well-architected single-page application hosted on Cloudflare Pages with a Supabase PostgreSQL backend containing 7,277 hadiths. The site demonstrates strong fundamentals in semantic HTML, security headers, and mobile responsiveness.

The primary concerns are missing asset files (OG image, PWA icons, favicons), some gaps in structured data coverage, and inline CSS that prevents browser caching. The security posture is excellent with comprehensive CSP, HSTS preload, and proper CORS configuration.

Critical Issues Requiring Immediate Action

Critical: Missing OG Image
All pages reference og-image.png which does not exist in the repository. Social sharing on Facebook, Twitter/X, LinkedIn, and WhatsApp will show a broken or blank preview. Create a 1200x630px branded image immediately.
Critical: Missing PWA & Favicon Icons
icon-192.png, icon-512.png, favicon-32.png, favicon-16.png, and apple-touch-icon.png are referenced but do not exist. This breaks PWA install, browser tabs, and mobile home screen icons.
Critical: CMS Page Uses Default Password
The cms.html admin page currently uses the SHA-256 hash of "password" as the default credential. This must be changed before deployment to production.

Inline CSS prevents caching
All pages embed ~20-30KB of CSS inline in <style> blocks. This means the CSS is re-downloaded on every page load. Extracting to an external stylesheet with a long cache TTL would improve repeat-visit performance.

3.2 Image Optimization

  • SVG icons used inline (zero HTTP requests)
  • Footer logo uses loading="lazy"
  • Width/height attributes set on images (prevents CLS)
  • Missing: WebP format for logo images
  • Missing: OG image, favicon, and PWA icon files

3.3 Font Loading Strategy

  • Preconnect to fonts.googleapis.com
  • Preconnect to fonts.gstatic.com (crossorigin)
  • Google Fonts API with display=swap
  • System fonts as primary body font (Inter fallback)
  • Arabic font (Noto Naskh Arabic) only loaded when needed
