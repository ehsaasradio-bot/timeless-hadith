Supabase PostgreSQL
Supabase Platform · PostgREST API · 7,277+ rows
Primary database at dwcsledifvnyrunxejzd.supabase.co. Stores all hadith records with 15 columns including tsvector for full-text search. Accessed directly from frontend via REST API with anon JWT key.
Database
🔐
Supabase Auth
Supabase Auth v2 · JWT HS256 · 10-year key
Handles user authentication via Google OAuth redirect and email magic links. Sessions stored in localStorage using th_supabase_session key. Replaces legacy Firebase Auth.
Authentication
🔥
Firebase Auth (Legacy)
Firebase JS SDK v10.14.1 (CDN — pinned)
Legacy authentication system loaded as ES module from gstatic.com CDN. Uses Google Identity popup flow. Scheduled for removal — being replaced by Supabase Auth. Currently coexists.
Auth (Deprecated)

OpenAI API
Model: gpt-4o-mini · max_tokens: 420 · temp: 0.3
Powers the AI Search feature via Cloudflare Edge Function. Implements a RAG pipeline: query → Supabase ilike search (top 5 hadiths) → GPT-4o-mini prompt → grounded answer. API key stored server-side in Cloudflare env.
AI / ML

Cloudflare Pages
Connected April 2026 · Auto-deploy on push to main
Primary hosting platform replacing GitHub Pages. Serves all static assets via Cloudflare's global edge network. Supports Pages Functions (serverless) for backend API routes. Zero-cost on free tier.

Workers Runtime · Route: /api/ai-search (POST)
Serverless edge functions running at Cloudflare's global PoPs. Handles the AI search pipeline — receives query, searches Supabase, calls OpenAI, returns JSON. No Node.js runtime needed.

Dependency Health & Risks
DependencyStatusRisk / Note
Firebase Auth SDK v10.14.1 ⚠ Legacy Loaded from CDN without version check. Coexists with Supabase Auth — creates dual auth systems. Should be removed once Supabase Auth is fully verified.
Supabase Anon JWT ⚠ Long-lived 10-year expiry (2026–2036). Embedded in 4 public files and in public GitHub repository. Cannot be rotated without updating all files simultaneously.
Google Fonts (googleapis.com) ⚠ No SRI Loaded without Subresource Integrity (SRI) hash. If Google's CDN were compromised, malicious CSS could be injected. Low real-world risk but a best-practice gap.
Cloudflare Beacon (beacon.min.js) ⚠ No SRI External script loaded without SRI. Analytics token exposed in source. Low sensitivity but worth noting as a dependency risk.
OpenAI API (gpt-4o-mini) ✓ Secure API key stored server-side in Cloudflare environment variables — never exposed to frontend. Correctly implemented.
Cloudflare Pages + Functions ✓ Healthy Managed platform with automatic updates. No version pinning needed. SLA backed by Cloudflare.
Vanilla JS / CSS (no framework) ✓ Zero dependency No npm packages, no node_modules, no supply chain risk for the core frontend. Excellent security posture for a static site.
