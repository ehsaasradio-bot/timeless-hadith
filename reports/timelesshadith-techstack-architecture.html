System overview

Timeless Hadith is a dual-stack platform — a fast static site for Islamic content discovery, and a full-featured e-commerce shop. Both deploy to Cloudflare Pages from a single GitHub repository and share the same Supabase PostgreSQL database, but they are architecturally independent.

Static Hadith Site
Vanilla JavaScript SPA served as static HTML from Cloudflare Pages. No build step. Data fetched client-side directly from Supabase PostgREST. All 7,277 hadiths, categories, bookmarks, search, and Urdu toggle.
Vanilla JS Cloudflare Pages Supabase REST
E-Commerce Shop
Next.js 15 App Router application with full edge runtime. All API routes run on Cloudflare's edge network. Stripe payments, inventory management, admin dashboard, and customer analytics.
Next.js 15 Edge Runtime Stripe Supabase
Database
Single Supabase PostgreSQL instance hosting both the hadith corpus and the full shop schema. RLS enforced on all tables. Anon key for public reads, service role key only on the server.
PostgreSQL RLS Triggers
Deployment
Both apps deploy to Cloudflare Pages on every push to main. The static site deploys directly; the shop goes through a Cloudflare Pages build using @cloudflare/next-on-pages. Zero manual deployment steps.
Cloudflare Pages Auto-deploy Edge CDN

API routes — admin-app

All routes use edge runtime. Admin routes require a valid HMAC-SHA256 session cookie.

Public routes

POST /api/checkout Create Stripe Checkout Session. Fetches prices server-side (never trusts client). Creates pending order row linked to Stripe session ID.
POST /api/webhook Stripe webhook receiver. Validates signature. Handles checkout.session.completed (mark order paid) and checkout.session.expired. Idempotent.
GET /api/shop/products List products with pagination. Supports filters: category, search (full-text), sort (popular/newest/price/rating). Returns count for pagination.
GET /api/shop/products/[slug] Single product detail with images array and current inventory level.
GET /api/shop/categories All active categories. Cached 5 minutes at CDN edge (s-maxage=300, stale-while-revalidate=3600).
GET·POST /api/shop/reviews GET returns approved reviews for a product. POST submits a new review (pending admin approval).
POST /api/shop/newsletter Subscribe email to newsletter. Upserts row (safe to call multiple times for same email). Records interest tags and source.

Admin routes (HMAC auth required)

POST·DEL /api/admin/login POST: verifies ADMIN_SECRET, sets httpOnly session cookie. DELETE: clears cookie (logout).
CRUD /api/admin/products Full product management. GET lists all, POST creates, PUT updates (include id), DELETE removes. Also supports inventory adjustment via initial_quantity / quantity_on_hand fields.
CRUD /api/admin/categories Category management. Auto-generates slug from title. Full CRUD with slug uniqueness enforcement.
GET·PUT /api/admin/orders GET with status/date filters and pagination. PUT to transition order through status pipeline: pending → confirmed → processing → shipped → delivered.
GET·PUT·DEL /api/admin/reviews Moderation queue. GET returns pending/approved reviews. PUT to approve. DELETE to remove.
GET /api/admin/stats Dashboard summary — total orders, revenue, product count, newsletter subscribers, low-stock alert count.
GET /api/admin/analytics Period revenue, order counts, daily breakdown for charting. Supports from/to date params and period-over-period comparison.
CRUD /api/admin/coupons Discount code management. Supports percent and fixed-amount types, min order values, use limits, and expiry dates.
GET /api/admin/customers Customer list grouped by email. Search by name/email. Drill into a customer's full order history.
GET·PUT /api/admin/settings Single-row shop configuration — name, tagline, currency, free shipping threshold, low-stock threshold, social links.

Authentication — how it works

Admin session (HMAC-SHA256)

Stateless — no database session table, no Redis. The session token is deterministically derived from the ADMIN_SECRET at login time and stored in an httpOnly cookie.

// lib/admin-auth.ts
deriveSessionToken(secret):
  HMAC-SHA256(secret, "th-admin-session-v1")
  → base64url string

isAdminAuthenticated(request):
  cookie = request.cookies.get('th_admin_session')
  expected = deriveSessionToken(ADMIN_SECRET)
  return timingSafeEqual(cookie, expected)
  // constant-time → prevents timing attacks

Changing ADMIN_SECRET in Cloudflare Pages env vars instantly invalidates all existing sessions.

Supabase key split

Two Supabase keys are used, strictly separated by context.

KeyWhere usedPermissions
anonFrontend JS, public API routesRespects RLS — read-only public data
service_roleAdmin API routes only (server-side)Bypasses RLS — full read/write access

The service_role key is never exposed to the browser or committed to git. It only exists in Cloudflare Pages environment variables and local .env.local (gitignored).

Notable implementation patterns

PatternWhereHow it works
Parallel paginationsupabase-data.jsSupabase PostgREST caps responses at 1,000 rows. JS fires multiple parallel requests with different offset ranges, then merges results to load all 7,277 hadiths.
Stateless admin authlib/admin-auth.tsHMAC-SHA256 derives a deterministic token from ADMIN_SECRET. No session table needed. Rotating the secret immediately invalidates all sessions globally.
Server-side pricingapp/api/checkoutCheckout route ignores client-submitted prices entirely. It fetches prices from Supabase before creating the Stripe session — a customer cannot manipulate what they're charged.
Idempotent webhooksapp/api/webhookStripe may deliver the same webhook event multiple times. The handler checks if the order is already marked paid before writing, making it safe to replay.
Inventory triggersSupabase SQLA PostgreSQL trigger fires on every shop_order_items insert to atomically decrement quantity_on_hand and write an audit row to shop_inventory_log.
Sequential order numbersSupabase SQLA trigger generates TH-YYYY-#### formatted order numbers on insert, readable by staff and safe for customer communications.
Theme flash preventionjs/theme-init.jsA tiny script injected before any CSS in <head> reads localStorage and sets data-theme on <html> synchronously, preventing the white flash on page load for dark-mode users.
Logo probingjs/app.jsChecks Supabase Storage hourly for a custom logo asset. If found, swaps the default logo. Allows brand updates without a code deploy.
Edge runtime everywhereAll API routesEvery API route declares export const runtime = 'edge'. All logic uses Web Crypto, fetch(), and Headers — no Node.js APIs. Executes in under 10ms globally.
Single-row settings tableshop_settingsA single PostgreSQL row (id=1) stores all shop configuration. Simpler than a key-value table, fully typed, and queryable as a regular row.
