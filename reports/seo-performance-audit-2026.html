⚙️ Script Loading Strategy

🚫
4 JS files load without defer/async
supabase-data.js, app.js, supabase-auth.js, and ai-search.js are loaded synchronously at the bottom of the HTML body. They block HTML parsing and delay rendering. Adding defer to all 4 would reduce Total Blocking Time significantly.
Impact: High — affects TBT and LCP
⚠️
No JavaScript minification
All JS files are served as unminified source with comments and whitespace. 79.5 KB unminified could be reduced to ~40–45 KB minified, cutting JS transfer time by ~45%.
Impact: Medium — affects parse time
⚠️
Google Fonts blocks rendering
The Noto Naskh Arabic Google Font loads as a render-blocking stylesheet. Adding font-display=swap to the font URL and preloading the WOFF2 file would prevent the flash of invisible text (FOIT) and improve FCP.
Impact: Medium — affects FCP
✅
Cloudflare CDN providing fast TTFB
All pages respond in under 400ms TTFB. Cloudflare's global edge network is serving cached content efficiently. Cache-Control headers are set (max-age=14400) for CSS, JS, and images.
Status: Working well — maintain this

Prioritised Recommendations

🚨 Fix Sitemap Slugs — Replace book-N with SEO-Friendly URLs

🔴 High Impact

97 of your 102 sitemap URLs use generic numeric IDs (?cat=book-1) instead of the descriptive slugs already used on live pages (?cat=prayer-worship). This prevents Google from understanding the content of category pages. Regenerating the sitemap with live slugs and adding unique title/meta per category could dramatically improve search rankings for topic-specific queries.

🚨 Add defer to All Script Tags — Reduce Total Blocking Time

🔴 High Impact

Adding defer to supabase-data.js, app.js, supabase-auth.js, and ai-search.js would allow the HTML to parse and render before JavaScript executes. Estimated improvement: 100–200ms reduction in Total Blocking Time, pushing performance score to 80+. This is a single-character change per script tag.

🚨 Convert PNG Images to WebP/AVIF

🔴 High Impact

All 16 PNG images should be converted to WebP (or AVIF for supporting browsers). The largest PNG (101 KB logo) would drop to ~20 KB in WebP — a 80% reduction. Total estimated image savings: 250–300 KB. Use the SVG logo (581 bytes) wherever possible — it's already available and infinitely scalable.

⚠️ Minify JavaScript Files

🟡 Medium Impact

Running all 4 JS files through a minifier (esbuild, terser, or Cloudflare Worker transforms) would reduce 79.5 KB of JS to approximately 40–45 KB — a ~45% bandwidth reduction. Add source maps for debugging. Consider bundling into a single file to reduce round trips.

⚠️ Increase Asset Cache TTL to 1 Year (with versioning)

🟡 Medium Impact

JS and CSS currently cache for 4 hours. Since supabase-data.js?v=2 already uses a query string version, it's safe to set max-age=31536000, immutable. Returning visitors would load the site from cache on subsequent visits, improving repeat-visit performance dramatically.

⚠️ Add Structured Data: BreadcrumbList + CollectionPage on Category Pages

🟡 Medium Impact

Category pages currently have only a generic WebSite schema. Adding CollectionPage + BreadcrumbList JSON-LD to each category page would enable richer Google SERP results (breadcrumb display) and better semantic understanding of the hadith collection structure.

ℹ️ Trim Meta Descriptions to 155 Characters

🔵 Low Effort

About (170 chars) and Categories (161 chars) descriptions exceed Google's display limit of ~155–160 characters. Google will truncate them with "…" in search results, cutting off key phrases. Trim each to under 155 characters for complete SERP display.

ℹ️ Add font-display=swap to Google Fonts URL

🔵 Low Effort

Adding &display=swap to the Google Fonts URL prevents invisible text while the font loads (FOIT), improving perceived performance and FCP. Add <link rel="preload"> for the Arabic font WOFF2 file to eliminate the loading delay entirely.

✅ Quick Win: Add Service Worker for Offline Caching

🟢 Big Opportunity

The PWA manifest is in place but there's no Service Worker. Adding a basic Service Worker with a cache-first strategy for static assets would: (1) make the site work offline, (2) dramatically improve repeat-visit load times by serving assets from cache, and (3) unlock PWA installability fully. A Workbox-based Service Worker can be implemented in under 50 lines.
