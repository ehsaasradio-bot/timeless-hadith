04 Manual Actions Remaining
⚠️
The new CMS login will not work until you set the environment variables in Cloudflare Pages. The site will deploy and load correctly, but attempting to log into the CMS will return an error until Action 1 is completed.
🔑
Action 1 — Cloudflare Env Vars (URGENT)
Set these two environment variables in Cloudflare Pages so the new CMS auth system can validate your password and sign session cookies.
Pages → timeless-hadith → Settings → Environment Variables
ADMIN_PASSWORD_HASH = <SHA-256 of your password>
TH_SESSION_SECRET = <any 40+ char random string>
Generate hash at: emn178.github.io/online-tools/sha256.html
📧
Action 2 — DMARC + SPF (H-04)
Add a DMARC DNS record and tighten SPF from ~all to -all to prevent email spoofing from your domain. Full instructions in the guide file.
Cloudflare DNS → Add TXT record
Name: _dmarc.timelesshadith.com
Value: v=DMARC1; p=quarantine; rua=mailto:dmarc@timelesshadith.com
Full guide: reports/dns-dmarc-setup.md
🔄
Action 3 — Supabase Key Rotation (C-03)
The anon key was deferred. Rotate it when you're ready to invalidate the old key and update the frontend config.
Supabase → Settings → API → Regenerate anon key
Update NEXT_PUBLIC_SUPABASE_ANON_KEY in Cloudflare Pages
Also update any hardcoded references in js/supabase-data.js
Also set JWT expiry to 30 days in Auth → Settings (H-01)
