Findings

High Font loading — no preload hints

Inter and Noto Kufi Arabic are render-critical but not preloaded. Add <link rel="preload" href="/fonts/inter-latin.woff2" as="font" type="font/woff2" crossorigin> for the 3 most-used font files.

Medium Gulzar Arabic font is 209 KB

This Urdu/Arabic decorative font is large. Consider subsetting to the Unicode ranges actually used, which could cut it to ~80 KB.

Medium data.js (33 KB) is loaded but may be unused

The static data.js file exists alongside the Supabase data layer (supabase-data.js). If supabase-data.js fully replaces it, remove data.js to save 33 KB.

Nice prayer-times.html has ~600 lines of inline CSS + JS

This page uses inline styles/scripts instead of the shared css/styles.css. Works but adds 20 KB of non-cacheable CSS per visit.

Pass Strengths

All scripts use defer. Single CSS file. WebP images with PNG fallback via <picture>. Service worker registered. Cloudflare CDN with edge caching. HTTP/3 + Brotli compression. Small DOM (~200 elements). font-display: swap on all fonts. Cache headers configured per file type.

Critical SEO Issues

Critical Blog post slug pages don't exist

12 blog cards link to URLs like /blog/sunnah-daily-habits, /blog/quran-hadith-relationship, etc. These all return 404. This creates 12 broken internal links and wastes significant link equity. Either create these pages or remove the links.

High blog.html and prayer-times.html missing canonical tags

Without canonical tags, search engines may treat URL variants (with/without trailing slash, query params) as duplicate content.

High category.html H1 renders as "Loading..." to crawlers

Google sees the pre-render HTML which has <h1 id="cat-title"></h1> (empty) and the JS-rendered "Loading hadiths..." text. Consider server-side rendering or adding a fallback H1 in the HTML.

Medium Thin content on homepage and categories page

index.html has only 151 words of body text; categories.html has 62 words. Google may classify these as thin content. Add introductory paragraphs (300+ words) to improve ranking potential.

Pass Strengths

Comprehensive sitemap.xml with 300+ URLs including parameterized category pages. Clean robots.txt. No duplicate titles. Strong schema markup on 7/10 pages (WebSite, Organization, FAQPage, CollectionPage, ItemList). Good URL structure. No redirect chains.

Critical Security Findings

Critical CSP connect-src blocks 4 essential APIs

The Content-Security-Policy connect-src directive only allows 'self', Supabase, the Worker, and Cloudflare Insights. It does NOT allow:

  • https://api.aladhan.com — Prayer times (broken on production)
  • https://api.metals.live — Gold/silver prices for Zakat calculator (broken)
  • https://nominatim.openstreetmap.org — Reverse geocoding (broken)
  • https://api.open-meteo.com — Weather widget (broken)
  • https://fonts.googleapis.com — Google Fonts on prayer-times.html (style-src)

Fix: Update _headers file to add these domains.

High CSP uses unsafe-inline for scripts

script-src 'self' 'unsafe-inline' weakens XSS protection significantly. Migrate to CSP nonces or hashes for inline scripts.

Medium Firebase/Google remnant in CSP

script-src still includes https://accounts.google.com from the old Firebase Auth. Remove it since auth has migrated to Supabase.

Medium Permissions-Policy blocks geolocation

geolocation=() blocks the Geolocation API entirely. But prayer-times.html uses navigator.geolocation for local prayer times. Change to geolocation=(self).

Pass Strengths

No exposed secrets or API keys (Supabase anon key is intentionally public/read-only). Admin page protected by Supabase Auth with 2FA. Strong security headers overall. No mixed content. No known JS library vulnerabilities (vanilla JS, no frameworks). Cloudflare DDoS protection.

6. Accessibility Audit (WCAG 2.2 AA)

Pass Color contrast

All color pairs exceed WCAG AA minimum (4.5:1). Light theme: ink #1d1d1f on #ffffff = 18:1. Dark theme: ink #f0f4ff on #060c1a = 17.5:1. Accent colors on surfaces also pass.

Pass Semantic HTML

Proper use of <main>, <nav>, <header>, <footer>, <article>, <section> across all pages. ARIA landmarks and roles present. Skip-to-content link implemented correctly.

Pass Keyboard navigation & focus states

Visible focus indicators on all interactive elements (buttons, links, inputs). :focus-visible with outline: 2px solid var(--accent). Respects prefers-reduced-motion: reduce.

Medium Theme toggle button too small (26px)

WCAG 2.2 requires 44x44px minimum tap target. The theme toggle is only 26x26px. Increase to at least 44x44px or add padding to the click area.

Nice Avatar alt text inconsistent

6 pages have alt="" on the nav avatar image; bookmarks.html correctly uses alt="User avatar". Standardize to alt="User avatar" across all pages.

7. Mobile Audit

Critical blog.html and prayer-times.html have no mobile navigation

These two pages use a simplified nav bar that hides links with display:none on mobile. There is no hamburger menu or drawer. Users on mobile have NO way to navigate to other pages except the logo (home) link.

Pass Viewport & responsive

All pages have <meta name="viewport" content="width=device-width, initial-scale=1.0">. Two responsive breakpoints at 960px and 680px. No horizontal overflow issues. Font sizes appropriate on mobile.

Pass Touch targets

Primary interactive elements (buttons, cards, nav links) meet the 44x44px minimum. Only the theme toggle (26px) and some secondary elements are undersized.

8. Error Detection

Critical 12 broken blog post links (404s)

Every blog card in blog.html links to a slug page that does not exist: /blog/sunnah-daily-habits, /blog/quran-hadith-relationship, /blog/fasting-sunnah-guide, etc. All return 404.

Critical AI search returns server_error

POST to /api/ai-search returns {"error":"server_error"} because the Cloudflare Worker's OpenAI API key is expired (401 Incorrect API key).

Critical 4 features broken by CSP

Prayer times, Zakat calculator gold/silver prices, weather widget, and geolocation are all silently failing because CSP blocks their API domains.

Pass No broken images, CSS, or internal page links

All image src paths resolve. All CSS loads. All internal page-to-page href links point to existing files (except the blog slugs noted above).

9. Conversion / UX Audit

Pass CTA clarity

Clear primary CTAs on homepage: "Browse all categories", AI search input, featured hadith cards. Good visual hierarchy and scannable layout.

Pass Trust signals

Source attribution on every hadith (Sahih al-Bukhari #xxx, Narrated by...). Explicit mention of "six major hadith collections." Professional design builds credibility.

High AI search error destroys trust

The AI search prominently features "Ask AI about any hadith..." with example queries. When users try it and get "server_error", it severely damages trust and credibility. This is the #1 UX issue.

Medium Blog cards lead to 404

Users clicking blog cards hit dead ends. This increases bounce rate and frustrates engaged visitors who wanted to read more.
